Admin / back-end example

This is the simple idea behind the admin side.

Staff should not edit the website code to change wines, prices, or stock.

Instead, they update a simple admin system such as Airtable or Google Sheets. The website then shows the approved result.

So in plain English: Airtable or Google Sheets would be the place where staff edit wines. Shopify would mainly be the customer-facing shop and checkout.

The information is split into separate tables so everything is easier to manage:

  • Catalogue = what the wine is
  • Pricing = what it costs / sells for
  • Stock = how many bottles and whether it is active
  • Media = pictures
  • Publishing = what the customer sees

Best practical version 1:

  • Airtable or Google Sheets = admin system
  • Shopify = live website and checkout
  • A small sync step = sends approved wines from the admin system to the website
