Admin / back-end example

This is the simple idea behind the admin side.

Staff should not edit the website code to change wines, prices, or stock.

Instead, they update a table-based admin system. The website then shows the approved result.

The information is split into separate tables so everything is easier to manage:

  • Catalogue = what the wine is
  • Pricing = what it costs / sells for
  • Stock = how many bottles and whether it is active
  • Media = pictures
  • Publishing = what the customer sees
