Build this app from scratch

This page is for the first build. It is written for someone with limited IT training who needs to set up the app from the beginning and get it working.

Step 1. Decide the four basic details

Before you start clicking around Microsoft 365, decide these four things.

  1. Choose which account will own the Microsoft Form.
  2. Choose which inbox will receive the staff emails first.
  3. Choose which shared mailbox will be used later.
  4. Choose where the Excel lookup workbook will live.

Step 2. Create the Microsoft Form

The easiest way to do this first build is not to type every question in by hand. Use the prepared Word file below. This is the file to edit if needed, and then upload into Microsoft Forms.

  1. Open Microsoft Forms.
  2. Select Quick Import.
  3. Select Upload from this device.
  4. If you need to change the wording first, edit student_enquiry_form_import.docx.
  5. Choose student_enquiry_form_import.docx from the local folder.
  6. When Forms asks, select Form.
  7. Wait for the import to finish, then select Start review.
  8. Check that the imported form contains the title, introduction text, and the expected questions.
  9. Make sure the introduction tells students they must be signed in and do not need to type username or email.
  10. Do not add a separate University username question in the preferred build.
  11. Do not add a separate forgot username question in version 1.
  12. Do not add a separate university email question unless testing later shows the flow cannot access the responder's email.
  13. Rename or tidy any question that imported imperfectly.
  14. After the questions are in place, go to the Stage of study question, open More settings for question, and select Add branching.
  15. Set the Undergraduate branch to go to Year of study.
  16. Set the Taught postgraduate branch to skip forward to Programme of study.
  17. Go to the Enquiry type question, open More settings for question, and select Add branching.
  18. Set the Other branch to go to Other enquiry type.
  19. Set all other enquiry-type options to skip forward to Message.
  20. Keep the branching moving forward only. Microsoft Forms branching works to later questions, not back to earlier ones.
  21. Open the form settings and set the form so only people in the organization can respond.
  22. In the same settings area, turn on recorded responder identity because that is what lets the flow use the responder's email for acknowledgements and verification.
  23. Set the thank-you message so it tells the student they will receive an acknowledgement email.

Step 3. Create the Excel lookup file

This workbook is used only to look up student details. It is not the main workflow tool.

  1. Open student_lookup_template.xlsx.
  2. Upload or save that workbook into OneDrive or SharePoint where the flow account can reach it.
  3. Open Excel Online and open that workbook.
  4. Confirm the workbook contains a table named StudentLookup.
  5. Confirm the first row contains the lookup columns.
  6. Add a small number of real or test rows first.
  7. Make sure UniversityEmailNormalized stays lowercase and unique.
  8. Make sure the Power Automate connection account can access the workbook.

Step 4. Start the Power Automate flow

This creates the automation that joins everything together.

  1. Open Power Automate.
  2. Select Create.
  3. Select Automated cloud flow.
  4. Name the flow Student enquiry intake - v1.
  5. In the trigger search box, search for Microsoft Forms.
  6. Select the trigger When a new response is submitted.
  7. Select Create.
  8. In the trigger card, choose the department form in the Form Id field.
  9. Select + New step.
  10. Search for Microsoft Forms and add Get response details.
  11. In Form Id, choose the same form again.
  12. In Response Id, select the dynamic value from the trigger.

Step 5. Add the student email and reference number

This part picks up the recorded responder email, derives the CIS username if needed, and creates the enquiry number.

  1. Select + New step and add a Compose action.
  2. Name it Compose_RecordedResponderEmail.
  3. In that compose action, use the Forms dynamic value for the responder's email.
  4. Select + New step and add another Compose action.
  5. Name it Compose_NormalizedResponderEmail.
  6. Use the local expression to convert that email to lowercase and trim spaces.
  7. If that responder email dynamic value does not appear, stop and decide whether to add a separate required University email address field before final testing.
  8. Select + New step and add a Compose action for the username.
  9. Name it Compose_SelectedUsername.
  10. Use the local expression so the flow keeps only the part before @.
  11. Select + New step and add a Compose action for the best email address.
  12. Name it Compose_SelectedStudentEmail.
  13. Use the local expression so the flow uses the recorded responder email as the student email.
  14. Select + New step and add a Compose action for the response ID.
  15. Name it Compose_ResponseId.
  16. Select + New step and add a Compose action for the reference number.
  17. Name it Compose_ReferenceNumber.
  18. Use the expressions in power_automate_expressions.html so the username, email, and reference number are all created correctly.

Step 6. Add the Excel lookup

This part asks Excel whether the recorded university email matches a known student record.

  1. Select + New step and search for Excel Online (Business).
  2. Add the action Get a row.
  3. Point it at the workbook you uploaded in Step 3.
  4. Choose the table StudentLookup.
  5. Set the key column to UniversityEmailNormalized.
  6. Set the key value to the normalized responder email compose output.
  7. Use version_1_flow_logic.html for the remaining lookup handling and fallback logic.

Step 7. Add the staff and student emails

This part sends one email to staff and one acknowledgement email to the student.

  1. Add the subject-line tags and suggested owner logic using version_1_flow_logic.html.
  2. Search for Office 365 Outlook and add Send an email (V2) for the staff email.
  3. Use staff_email_template.txt for the staff email subject and body layout.
  4. Add a second Outlook email action for the student acknowledgement.
  5. Use student_acknowledgement_email.txt for that message.
  6. Paste the local template wording into the Outlook actions inside Power Automate.
  7. Keep the wording simple and professional.
  8. Do not promise response times unless the department is ready to meet them.
  9. Save the flow.

Step 8. Test the app

Do not skip this. Test before moving the app into a shared mailbox.

  1. Submit a test response through the form and check the run history in Power Automate.
  2. Test a matched undergraduate enquiry.
  3. Test a matched taught postgraduate enquiry.
  4. Test an unmatched university email.
  5. Test an Other enquiry type.
  6. Test an extension or mitigating circumstances enquiry.
  7. Check that the reference number appears, the emails send, and the flow still works when lookup fails.

Step 9. Move to the shared mailbox

Only do this after the pilot inbox version is working properly.

  1. After the pilot works, change the recipient from the pilot inbox to the shared mailbox.
  2. If needed, change the send action to Send an email from a shared mailbox.
  3. Confirm mailbox permissions.
  4. Run the tests again.

Step 10. Stop here for version 1

Version 1 is ready once students can submit the form, staff receive one clear structured email, the student receives an acknowledgement with a reference number, and matched and unmatched lookups both behave properly.

Do not add case management, hard routing, or deep Banner integration yet.

After this first build is working, you can create a more reusable version for other teams to copy and edit.
