Build email app from scratch

This is the first-build page. Start by downloading the folder, edit the four files that matter, then keep the form short and use Banner export or the lookup workbook for stable student facts.

Step 1. Download the starter pack

Download the full folder first so you have the files in one place.

Download starter pack

Step 2. Edit only these files first

Do not edit everything. Start with these four files only.

  1. Edit student_enquiry_form_import.docx if you want to change the form wording before import.
  2. Edit student_lookup_template.xlsx and add real or test student rows, ideally from Banner export.
  3. Edit staff_email_template.txt if you want different wording for staff.
  4. Edit student_acknowledgement_email.txt if you want different wording for students.

If someone else is doing the live build, this is the folder to give back to them.

Step 3. Decide the four basic details

Before you start clicking around Microsoft 365, decide these four things.

  1. Choose which account will own the Microsoft Form.
  2. Choose which inbox will receive the staff emails first.
  3. Choose which shared mailbox will be used later.
  4. Choose where the Excel lookup workbook will live.

Step 4. Create the Microsoft Form

Do not type the whole form by hand. Upload the prepared Word file into Microsoft Forms.

  1. Open Microsoft Forms.
  2. Select Quick Import.
  3. Select Upload from this device.
  4. Choose student_enquiry_form_import.docx from the local folder.
  5. When Forms asks, select Form.
  6. Wait for the import to finish, then select Start review.
  7. Check the imported title, introduction, and questions.
  8. Keep the form signed-in only and do not add separate username or email questions unless testing later forces that.
  9. Add branching so Other shows Other enquiry type and all other enquiry types go straight to Message.
  10. Keep branching moving forward only.
  11. Open form settings and make sure only people in the organization can respond.
  12. Turn on recorded responder identity so the flow can use the student's email for lookup and acknowledgement.
  13. Do not ask for stage, year, or programme in the preferred build.
  14. Only add one checkpoint field later, such as Programme of study, if testing shows the lookup is unreliable.
  15. Set the thank-you message so it tells the student they will receive an acknowledgement email.

Step 5. Create the Excel lookup file

This workbook is used only to look up student details. It is not the main workflow tool or the main staff interface.

  1. Open student_lookup_template.xlsx.
  2. Save that workbook into OneDrive or SharePoint where the flow account can reach it.
  3. Open Excel Online and open that workbook.
  4. Confirm the workbook contains a table named StudentLookup.
  5. Prefer to populate the workbook from Banner export or another trusted student-record source.
  6. Add a small number of real or test rows first.
  7. Make sure UniversityEmailNormalized stays lowercase and unique.
  8. Make sure the Power Automate connection account can access the workbook.

Step 6. Create the Power Automate flow

This is the basic flow shell. Once this is in place, you will add the logic and the emails.

  1. Open Power Automate.
  2. Select Create.
  3. Select Automated cloud flow.
  4. Name the flow Student enquiry intake - v1.
  5. In the trigger search box, search for Microsoft Forms.
  6. Select the trigger When a new response is submitted.
  7. Select Create.
  8. In the trigger card, choose the department form in the Form Id field.
  9. Select + New step.
  10. Search for Microsoft Forms and add Get response details.
  11. In Form Id, choose the same form again.
  12. In Response Id, select the dynamic value from the trigger.

Step 7. Finish the Power Automate flow

Add the logic in this order: identity, reference number, Excel lookup, staff email, student email.

  1. Keep power_automate_expressions.html open while you do this step.
  2. Add these five Compose actions in this order: Compose_RecordedResponderEmail, Compose_NormalizedResponderEmail, Compose_SelectedUsername, Compose_SelectedStudentEmail, Compose_ReferenceNumber.
  3. Use the helper page to paste in the expressions for the normalized email, username, student email, and reference number.
  4. If the Forms responder email field is missing, stop here and decide whether you need a separate required university email question on the form.
  5. Add Excel Online (Business) - Get a row.
  6. Point it at the workbook from Step 5, choose StudentLookup, set the key column to UniversityEmailNormalized, and set the key value to the normalized responder email.
  7. Keep version_1_flow_logic.html open while you finish the rest of the flow.
  8. Add the suggested owner and subject-line tags from the helper page.
  9. Add one Outlook action for the staff email and one Outlook action for the student acknowledgement.
  10. Use staff_email_template.txt for the staff email and student_acknowledgement_email.txt for the student email.
  11. If the lookup fails, continue anyway and let staff check Banner or the student record manually.
  12. Do not promise response times unless the department is ready to meet them.
  13. Save the flow.

Step 8. Test the app

Do not skip this. Test before moving the app into a shared mailbox.

  1. Submit a test response through the form and check the run history in Power Automate.
  2. Test one matched student and one unmatched student.
  3. Test one undergraduate enquiry and one taught postgraduate enquiry.
  4. Test one Other enquiry and one normal enquiry.
  5. Check that the reference number appears, the emails send, and the flow still works when lookup fails.

Step 9. Move to the shared mailbox

Only do this after the pilot inbox version is working properly.

  1. After the pilot works, change the recipient from the pilot inbox to the shared mailbox.
  2. If needed, change the send action to Send an email from a shared mailbox.
  3. Confirm mailbox permissions.
  4. Run the tests again.

Step 10. Stop here for version 1

Version 1 is ready once students can submit the form, staff receive one clear structured email, the student receives an acknowledgement with a reference number, and matched and unmatched lookups both behave properly.

Do not add case management, hard routing, or deep Banner integration yet.

After this first build is working, you can create a more reusable version for other teams to copy and edit.
