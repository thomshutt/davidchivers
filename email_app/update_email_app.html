Update email app

Use these steps when the app already exists and you want to change it safely.

Step 1. Decide what is changing

  1. Decide whether you are changing the form, the flow, the lookup file, the email templates, or the documentation.
  2. Try to change one area at a time.

Step 2. Update the Microsoft Form if needed

  1. Open the existing form.
  2. Change the wording, programme values, or enquiry types.
  3. Check that the branching still works after the change.
Open Forms Form question guide

Step 3. Update the lookup file if needed

  1. Open the Excel workbook.
  2. Update student rows or columns carefully.
  3. Keep UniversityEmailNormalized lowercase and unique.
  4. If you change the schema, make sure the flow still reads the same fields.
Open Excel Lookup schema

Step 4. Update the flow if needed

  1. Open the existing Power Automate flow.
  2. Change the logic, subject tags, suggested owner labels, or email content as needed.
  3. Keep the basic pattern the same: read the form, create a reference number, run lookup, send staff email, send acknowledgement.
Open Power Automate Flow logic Expressions

Step 5. Update the email templates if needed

  1. Update the staff email template if staff need clearer subject tags or body fields.
  2. Update the acknowledgement email if student wording needs to change.
  3. Keep both templates short and realistic.
Staff email template Student acknowledgement email

Step 6. Test after the change

  1. Run a matched enquiry.
  2. Run an unmatched enquiry.
  3. Check that the reference number still works.
  4. Check that staff and student emails still send correctly.
Build plan and test checklist

Step 7. Rebuild the docs if the documentation changed

  1. If you changed the Markdown docs, run build_docs.ps1.
  2. That will refresh the generated HTML and PDF files.
